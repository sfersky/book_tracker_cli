# Трекер прочитанных книг

Консольное приложение на Python для учёта прочитанных книг.

Проект сделан как учебная симуляция командной разработки: код разделён на модули, изменения можно выполнять в отдельных Git-ветках и объединять через Pull Request.

## Возможности приложения

Программа умеет:

1. Добавлять книгу.
2. Показывать все книги.
3. Показывать среднюю оценку.
4. Показывать статистику по авторам.
5. Удалять книгу.
6. Завершать работу.

Данные сохраняются в файл `books.json`.

## Поля книги

У каждой книги есть:

- автор;
- название;
- оценка от 1 до 5;
- дата прочтения в формате `ГГГГ-ММ-ДД`.

## Структура проекта

```text
book_tracker_cli/
├── main.py
├── storage.py
├── stats.py
├── models.py
├── README.md
└── .gitignore
```

## Назначение модулей

- `main.py` — точка входа, меню, ввод и вывод данных.
- `storage.py` — чтение, запись, добавление, поиск дубликатов и удаление книг из JSON.
- `stats.py` — расчёт средней оценки и статистики по авторам.
- `models.py` — класс `Book` и проверка данных книги.

## Как запустить

Нужен Python 3.10 или новее.

```bash
python main.py
```

или:

```bash
python3 main.py
```

## Пример Git-команд для выполнения кейса

Первый коммит:

```bash
git init
git add .
git commit -m "Создать начальную структуру проекта"
git branch -M main
```

Создание веток:

```bash
git checkout -b feature/stats-and-models
git add models.py stats.py
git commit -m "Добавить модель книги и статистику"

git checkout main
git checkout -b feature/storage
git add storage.py
git commit -m "Добавить хранение книг в JSON и проверку дубликатов. Closes #1"

git checkout main
git checkout -b feature/main-menu
git add main.py
git commit -m "Добавить консольное меню приложения"
```

Слияние веток после Pull Request или локально:

```bash
git checkout main
git merge feature/stats-and-models
git merge feature/storage
git merge feature/main-menu
git push
```

Граф коммитов для скриншота:

```bash
git log --oneline --graph --all
```
